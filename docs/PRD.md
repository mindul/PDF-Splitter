
# PDF Splitter 제품 요구 사양서 (PRD)

## 1. 프로젝트 개요
**PDF Splitter**는 사용자가 브라우저 환경에서 별도의 서버 전송 없이 PDF 파일을 업로드하고, 원하는 페이지만 선택하여 추출 및 다운로드할 수 있는 웹 애플리케이션입니다.

## 2. 핵심 목표
- **보안성 강화**: 모든 PDF 처리를 클라이언트 브라우저 내에서 수행하여 파일의 외부 유출 방지.
- **사용 편의성**: 직관적인 드래그 앤 드롭 및 클릭 인터페이스 제공.
- **즉시성**: 서버 업로드/다운로드 대기 시간 없이 즉각적인 처리.

## 3. 주요 기능 명세

### 3.1 PDF 파일 업로드
- **기능**: 사용자가 로컬 시스템의 PDF 파일을 애플리케이션으로 불러옵니다.
- **상세**:
  - Drag & Drop(드래그 앤 드롭) 지원.
  - 클릭하여 파일 탐색기를 통한 선택 지원.
  - PDF 파일 형식(`.pdf`)만 허용하며, 그 외 파일은 제한.
  - 단일 파일 처리 프로세스.

### 3.2 페이지 미리보기 및 선택
- **기능**: 업로드된 PDF의 각 페이지를 썸네일 형태로 확인하고 추출할 페이지를 지정합니다.
- **상세**:
  - **개별 선택**: 페이지를 클릭하여 선택/해제 토글.
  - **다중 선택 (Range Selection)**: `Shift` 키를 누른 채 클릭하여 마지막 선택된 페이지부터 현재 페이지까지 일괄 선택.
  - 선택된 페이지는 시각적으로 구별(테두리, 체크 표시 등).

### 3.3 PDF 추출 (Splitting)
- **기능**: 선택된 페이지들만 포함하는 새로운 PDF 파일을 생성합니다.
- **상세**:
  - `pdf-lib` 라이브러리를 활용하여 브라우저 메모리상에서 페이지 복사 및 병합.
  - 원본 파일의 손상 없이 새로운 문서 생성.

### 3.4 파일 다운로드
- **기능**: 생성된 PDF 파일을 로컬 저장소로 저장합니다.
- **상세**:
  - 자동 다운로드 트리거.
  - 파일명 형식: `{원본파일명}-extracted.pdf`. (예: `report.pdf` -> `report-extracted.pdf`)

### 3.5 초기화 (Reset)
- **기능**: 작업을 취소하고 초기 상태로 돌아갑니다.
- **상세**:
  - 현재 로드된 파일 및 선택된 페이지 정보 초기화.
  - 초기 업로드 화면으로 복귀.

## 4. 기술 스택
- **Frontend Framework**: Next.js 14+ (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **PDF Processing**: pdf-lib
- **Icons**: Lucide React

## 5. UI/UX 요구사항
- **Responsive Design**: 데스크탑 및 모바일 환경 지원.
- **State Feedback**:
  - 업로드 상태 (Drag Active 시 스타일 변경).
  - 페이지 선택 상태 명확한 시각적 피드백.
  - 추출 진행 중 로딩 인디케이터 표시 (필요 시).
